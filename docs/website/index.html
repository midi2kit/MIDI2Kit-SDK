Features

Everything you need to build MIDI 2.0 apps on Apple platforms

🔍

MIDI-CI Discovery

Automatically discover MIDI 2.0 capable devices on the network. Real-time event stream with async/await.

📦

Property Exchange

Read and write device properties via the PE protocol. Full transaction management with timeout and retry.

🔄

UMP Conversion

Bidirectional MIDI 1.0 ↔ 2.0 conversion. SysEx7, RPN/NRPN, and multi-packet reassembly built in.

📡

Responder API

Publish your own device on the MIDI-CI network with typed resources and connection policy filtering.

Code Examples

Clean, expressive APIs powered by Swift Concurrency

// デバイスの自動検出と情報取得
import MIDI2Kit

let client = try MIDI2Client(name: "MyApp")
try await client.start()

for await event in await client.makeEventStream() {
    switch event {
    case .deviceDiscovered(let device):
        print("Found: \(device.displayName)")

        if device.supportsPropertyExchange {
            let info = try await client.getDeviceInfo(from: device.muid)
            print("Product: \(info.productName ?? "Unknown")")
        }

    case .deviceLost(let muid):
        print("Lost: \(muid)")

    default: break
    }
}
// Property Exchange: GET / SET
import MIDI2Kit

let client = try MIDI2Client(name: "MyApp")
try await client.start()

// デバイスが見つかったら...
let device: MIDI2Device = /* discovered device */

// GET: デバイス情報を取得
let info = try await client.getDeviceInfo(from: device.muid)
print("Manufacturer: \(info.manufacturerName ?? "")")
print("Product: \(info.productName ?? "")")

// GET: リソース一覧
let resources = try await client.getResourceList(from: device.muid)

// GET: 任意のリソースを取得
let response = try await client.get("ChannelList", from: device.muid)

// SET: リソースを書き込み
let data = "{\"value\": 42}".data(using: .utf8)!
try await client.set("X-Custom", data: data, to: device.muid)
// Responder: 自分のデバイスをネットワークに公開
import MIDI2Kit

let responder = try MIDI2ResponderClient(name: "MyResponder")

// 型安全なリソースを登録
await responder.addResource("Temperature") {
    ComputedResource(getTyped: { _ in
        TemperatureReading(celsius: getCurrentTemp())
    })
}

// 接続フィルタリング
var config = MIDI2ClientConfiguration()
config.connectionPolicy = MIDI2ConnectionPolicy(
    allowedNames: [.prefix("iPad"), .contains("Pro")]
)

// 開始
try await responder.start()

Architecture

Modular design with clear dependency hierarchy

MIDI2Kit High-Level API MIDI2Client, MIDI2ResponderClient, MIDI2Device
↓
MIDI2CI Discovery CIManager, CIMessageParser
MIDI2PE Property Exchange PEManager, PETransactionManager
MIDI2Transport CoreMIDI CoreMIDITransport, SysExAssembler
↓
MIDI2Core Foundation MUID, DeviceIdentity, UMPBuilder, UMPParser, Mcoded7

Actor-Based Concurrency

All managers are Swift actors. All data types are Sendable. Full Swift 6 strict concurrency support.

Testable by Design

MockMIDITransport enables testing without hardware. 602+ tests across 77 test suites.

Zero External Dependencies

Only Apple frameworks (CoreMIDI, Foundation). No third-party packages required.

Getting Started

Up and running in 3 steps

1

Add the Package

In Xcode: File → Add Package Dependencies and enter the repository URL.

// Or add to Package.swift
dependencies: [
    .package(
        url: "https://github.com/midi2kit/MIDI2Kit-SDK.git",
        from: "1.0.0"
    )
]
2

Import and Create a Client

import MIDI2Kit

let client = try MIDI2Client(name: "MyApp")
try await client.start()
3

Discover Devices

for await event in await client.makeEventStream() {
    if case .deviceDiscovered(let device) = event {
        print("Found: \(device.displayName)")
    }
}
